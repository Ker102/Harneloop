# Human CLI Slice

**Goal:** Add a friendly Rich-powered CLI for users while preserving the explicit command API that agents and scripts use.

**Architecture:** The existing `argparse` commands remain the stable automation layer. The human CLI lives in `src/evorig/interactive.py` and calls the same lifecycle functions as the agent-facing commands. Setup defaults, artifact suggestions, and menu definitions live in `src/evorig/setup_flow.py` so they can be tested without terminal prompts.

**Tech Stack:** Python, Rich, PyYAML, `argparse`, and `unittest`.

## Decisions

- `evorig setup` starts a guided new-unit setup flow.
- Running `evorig` with no command opens the interactive menu when attached to a terminal.
- `evorig units` manages the local unit registry.
- `evorig settings` manages user preferences that affect agent behavior, validation, exports, and runtime defaults.
- Success criteria are optional during setup. The user can let the agent propose them or decide after a baseline attempt.
- Artifact choices are guided by validation preference. The user can choose best quality, visual/artifact-first, balanced, resource-efficient, or let the agent decide.
- The default unit path for interactive setup is `./units/<unit-id>`, and `units/` is ignored by git.
